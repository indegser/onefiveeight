# Core Reference Stack

These references define the engine grammar more than any single visual brand reference.

## Primary Sources

### 1. Refactoring UI

Role:

- Primary source for visual hierarchy, spacing scale discipline, contrast management, and layout balance.

Translate into engine rules:

- hierarchy first
- consistent spacing scale
- minimal color palette
- prefer font weight shifts before large font-size jumps

### 2. Gestalt Principles

Role:

- Grouping and layout perception model.

Translate into engine rules:

- use proximity to define groups
- use similarity for repeated components
- maintain continuity in reading and interaction flow
- preserve figure-ground clarity

### 3. Laws of UX

Role:

- Interaction and decision simplification model.

Translate into engine rules:

- reduce unnecessary choices
- keep targets large enough for fast acquisition
- prefer familiar patterns unless differentiation is justified
- keep cognitive chunking manageable

### 4. Grid Systems in Graphic Design

Role:

- Layout math for columns, rhythm, and alignment.

Translate into engine rules:

- use modular grid logic
- align components to stable columns
- preserve baseline rhythm for text-heavy surfaces
- avoid arbitrary edge alignment

### 5. Dieter Rams

Role:

- Product philosophy layer for restraint and clarity.

Translate into engine rules:

- good design is understandable
- good design is unobtrusive
- good design is aesthetic
- good design is minimal

## Current Repository Interpretation

For this repository, apply the stack in this order:

1. design tokens
2. layout rules
3. component grammar
4. UX laws and heuristics
5. aesthetic profile fit

The warm restrained editorial utility profile must remain compatible with these core references rather than replacing them.
