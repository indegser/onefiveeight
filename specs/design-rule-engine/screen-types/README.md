# Screen Type Contexts

Each screen design should be instantiated from a screen-type context template, then customized with task and product specifics.

Recommended process:

1. Pick the nearest screen type.
2. Copy its baseline fields into the design artifact.
3. Fill in task-specific goals, states, risks, and layout decisions.
4. Record unresolved questions rather than hiding them.

Screen-type context is intentionally separate from implementation details. A `table` screen may later bind to any component library or custom primitives.
